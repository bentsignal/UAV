# uav workflow

Use uav as shared project memory:

1. Start by checking project context and recent activity with `uav status`.
2. Check durable notes with `uav notes`, or `uav notes <query>` when looking for a specific remembered idea.
3. During work, record decisions, discoveries, blockers, and other context worth preserving.
4. When finishing or pausing, report what changed, what remains open, and validation status.
5. If uav needs a better command, schema, workflow, or policy, record an improvement request.

Store concise information that will matter later: goals, decisions, findings, blockers, follow-ups, validation results, and requests for uav to evolve.
