# uav workflow

Use uav as shared project memory:

1. Start by checking project context, recent activity, notes, tasks, and open requests.
2. During work, record decisions, discoveries, blockers, and other context worth preserving.
3. When finishing or pausing, report what changed, what remains open, and validation status.
4. If uav needs a better command, schema, workflow, or policy, record an improvement request.

Store concise information that will matter later: goals, decisions, findings, blockers, follow-ups, validation results, and requests for uav to evolve.
