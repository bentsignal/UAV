# uav

uav is shared memory for project work.

This repository is the uav project itself, so using this skill here is a
self-hosting case: use uav to coordinate work on uav, and record requests for
uav capability, schema, workflow, or policy changes as part of the project.

If this is a first run and the user mentions setting up the project, prioritize
the `setup` skill before using uav.

Run `uav help` for commands and `uav workflow` for current usage guidance.
